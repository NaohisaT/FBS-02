# リモートブラウザ操作・大量データ処理における絶対ルール

## 1\. 「十分な時間待つ」ことを必須とする

リモートブラウザでは：

* 描画に 5〜30秒

* シート貼り付けに 10〜60秒

* 行追加や大量スクロールは 1〜5分 かかる場合すらある

処理完了を待たずに次操作をすると、確実に破綻する  
したがって、操作後には必ず 最低 10〜20秒待機、  
 大量データ操作では 30〜90秒待機する。  
---

## 2\. 「画面が完全に安定してから」次に進む

以下の状態を“完全に安定”とみなします：

* 白画面（Blank）ではない

* シートがブレず、セル選択が表示されている

* スクロールが止まっている

* Name Box が更新されている

* シートの下部（行番号やメニューなど）が正常に表示される

安定を確認するまでは絶対に次操作をしない  
---

## 3\. 大量データ貼り付け時は「最低20秒」待つ

スプレッドシート側の処理が完了するまでは：

* 貼り付け中は画面が固まって見える

* 反応しなくてもバックグラウンドで処理中

* 無理に別操作をすると、貼り付けが中断される

最低20秒、長い場合は90秒まで待ち、  
 画面の動作が戻ってから次のステップへ

---

■ 画面スケールは常に 120%（推奨125%）

* メニューの当たり判定が広がり、クリック精度向上

* 操作ミス・サブメニュー閉じを大幅削減

* セル・タブ・メニューが視認しやすくなる

---

■ 操作は“必ず確認” → “クリック” → “確認”の3段階  
例：

1. メニューが見えるか確認

2. クリック

3. メニューが開いたことを確認

4. 次の項目をクリック

---

■ タブは“完全に見えている状態”でのみ右クリック

* 少しでも欠けていたら操作禁止

* 必要なら All Sheets（≡） から呼び出す

1. フォルダ一覧が見えるまで待機（5〜10秒）

1. 更新日時ヘッダをクリック

5\. データ範囲をコピー（安全版）

1. A2 をクリック

2. CTRL \+ SHIFT \+ ↓（描画反映待機 1秒）

3. CTRL \+ SHIFT \+ →（横方向も必ず選択）

4. CTRL \+ C

5. コピー完了まで 2〜3秒待機

※ 全選択が不完全なら、もう一度やり直す。

7\. 非表示シートを表示（メニュー操作の精密化）  
必ずメニュー操作：

1. 上部メニュー → 表示 を確認

2. クリック → サブメニューが開いたことを確認

3. 「非表示のシート」へカーソルを移動（0.5秒待機）

4. 『依頼報告』と『依頼報告\_顧客数』を順に表示

5. シート描画待機 10秒

※ クリックミスを防ぐため カーソル移動後0.5秒停止 を挟む。  
---

8\. 「依頼報告\_顧客数」へ貼り付け

1. タブをクリック（タブ完全表示確認）

2. A1 → CTRL+↓ → 空行へ移動

3. 貼り付けは必ず メニュー操作

   * 編集 → 特殊貼り付け → 値のみ貼り付け

4. 反映待機 10〜20秒

※ CTRL+SHIFT+V は不可（リモートだと信頼性が低い）。  
---

9\. 重複削除（フェイルセイフ完全版）

1. CTRL \+ A（失敗前提で扱う）

2. メニュー → データ → データクリーンアップ → 重複削除

3. ダイアログ確認：

   * 「◯◯まで拡張」が出たら 必ず押す

   * 「すべて選択」オフ

   * D・E・G のみチェック

4. 実行 → 完了ダイアログで OK

5. 処理完了待機 5〜10秒

---

10\. 「依頼報告」タブへ貼り付け

1. タブへ移動（描画 5秒待機）

2. A1 → CTRL+↓ → 空行へ

3. 編集 → 特殊貼り付け → 値のみ貼り付け

4. 完了待機 10秒

---

11\. 両シートを非表示に戻す

1. タブが 完全表示 されていることを確認

2. 右クリック → シートを非表示

3. 対象：

   * 依頼報告

   * 依頼報告\_顧客数

4. 必要なら All Sheets（≡）で呼び出し
